# NeuroLoom

NeuroLoom 是一个面向 `MLP`、`CNN` 和标准 `GPT` 架构大模型 (`Transformer`) 的开源神经网络运行回放解释器。

它不直接连接任意模型运行时，也不试图成为一个通用的计算图可视化平台。由于底层算法不可见，NeuroLoom 只负责读取受控产生的 `.loomtrace` 回放存档，把整场训练或推理还原为一个充满电影级物理光感的 2.5D 动画场景——在这里，一切节点发光、数据流水线、镜头移动以及数值监控全都精确对齐到了同一个时间帧上。

*其他语言版本：[English](README.md), [简体中文](README_zh.md)*

## 当前状态

该仓库已完成了 NeuroLoom V1 核心主线闭环，同时引进了高级 2.5D 物理渲染引擎与严苛的代码测试验证标准。

**已取得成就**：

- 加入极致逼真的 `meshPhysicalMaterial` 实现神经网络节点的真实透光、磨砂玻璃带折射效果发光组件。
- 整套场景深度集成电影级后期色彩管线（包括 `Cinematic Bloom`，`Chromatic Aberration` 镜头边缘色差以及沉浸式动态深空颗粒背景）。
- 完成了 `Replay-first`（回放第一）的核心工作流。囊括三个主要神经架构范式（`mlp`，`cnn`，`transformer`）以及它们的独立视觉案例探索模式。
- 系统内部置双环境架构设计：`Story Mode` (故事演示模式) 与 `Studio Mode`（探索解构模式）。
- **ONNX Runtime Web + WebGPU 浏览器端推理**——真实微型 ONNX 模型（MLP、CNN、Transformer）在浏览器内执行前向传播，激活值、特征图与注意力权重均来自真实计算而非合成数据。WebGPU 不可用时自动回退至 WASM。
- **隔离冻结工作流**——选中节点或边后按 `F` 键冻结，相机与检查器锁定至该元素，所有非相关节点与边近乎消失。按 `Esc` 解冻。
- 用 `Playwright` 搭起了端到端自动化 UI 与视觉回归基线（并引入带有语义确定性的 `data-testid` 机制）。
- 基于 TypeScript 的类型完整校验方案、ESLint 和 Prettier 代码风格加固。
- 成熟可靠的回放加载机制、Payload 安全反序列解析等架构引擎支持。

**尚未完全实装部分：**

## 为什么是神经回放？

NeuroLoom 关注的绝不是“画一张神经网络连接示意图长什么样”，而是**“在一次具体的执行步骤中，网络内部到底流过、发生了什么细节”**。

它把一次受控的数学模型推理/更新动作，切分为一组具备数学时序确定性的记录帧。在同一帧画面里，你可以同时捕获：
- **结构语义**：此刻有哪些网络节点、层次以及层级关联。
- **时间层语义**：我们所处于全流程中的哪一个步骤、小任务段，是推演 (`forward`) 还是反向传播梯形图 (`backward`)。
- **视觉映射**：哪里必须产生物理发光、什么流向的数据带有粒子响应、相机的核心聚集追踪焦段位置。
- **准确数字语义**：可以随时点击拉出的张量特征图、激活值、注意力矩阵。

NeuroLoom 并不是要把网络描出草图；它的职责是“演出”整个代码行为！

## 运行模式解析

### Story Mode (教学故事模式)
专为科普与展示而设计。随着系统设定好的一切场景角度与高光镜头标注，将复杂的 AI 前沿公式转为直观有情节的视觉叙事路径，适合入门演示者或教学活动。

### Studio Mode (工作室控制台模式)
专精供用户逐帧审查底层交互运行表现，提供了：
- 完全的局部回放进度拖拽及帧回退控制能力。
- 可以自由树状检索的节点树。
- 会向主显示区双向映射高光的专属数值检查检查器。
- 支持读取拖拽外部自定义产生的符合协议的 `.loomtrace` 生成压缩存档，支持保存场景物理截图 (`PNG`) 制备论文插图使用。

## 关于 `.loomtrace` 协议

NeuroLoom v1 唯一的指定通信准入协议即 `.loomtrace` 归档封包格式。所有行为都需要提供这样一份无二义性的 zip 打包清单描述受控的全部内容。

- `manifest.json` （控制源定义）
- `graph.json` （结构表谱）
- `timeline.ndjson` （带有时间戳轴的所有步）
- `payload/` （内部记录大型特征值及数组等数据包）
- `narrative.json`

这彻底免除了通用模型引擎转出时的难以调试的抽象临时特征猜测等错误。

## 如何参与及启动在你的电脑上

必须配置要求：
- `Node.js 22+`
- `pnpm 10+`

第一步安装依赖包：
```bash
pnpm install
```

初始化内部预设模拟测试用的 3 大经典例子痕迹文件 (\`.loomtrace\`) ：
```bash
pnpm generate:traces
```

唤起沉浸式的高级三维工作台交互前端测试：
```bash
pnpm dev
```
